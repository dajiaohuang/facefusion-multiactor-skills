# Plan Refinement Patterns

Use this phase after role merge and source binding are already in place.

Phase goals:

- add optional per-shot operations when requested
- build `plan.json`
- review preview/final task structure
- approve previews or retry failed tasks

Ask:

- which shots need optional pipeline additions
- whether those additions apply to all shots or selected shot ids
- whether preview outputs should be approved or revised

Do not:

- restart from cast collection
- redo reference merge unless the user explicitly changed role binding
