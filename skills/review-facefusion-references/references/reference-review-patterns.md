# Reference Review Patterns

Use this phase after the initial draft exists and before final plan build.

Phase goals:

- detect target-face clusters
- merge clusters into final roles
- bind or defer source faces

Ask:

- which clusters should merge
- whether prefilled source matches should stay or change
- which roles are still waiting for source input

In UI mode:

- render `reference-view.html`
- let the user refine merge/source decisions there

In no-UI mode:

- summarize clusters, shots, prefills, and missing sources in text
- continue the same decision loop conversationally
